## 🚀 AFC-7 智能飞控板 - 底层环境一键安装与验证指南
**平台架构**：AFC-7新一代智能飞控
**文档版本**：V1.01.260505 
**开源平台**：[zqhkd/AFC-7Published: AFC-7智能飞控软件开发平台](https://github.com/zqhkd/AFC-7Published)
**日   期**: 2026-05-05
**作   者**: 曾庆华

欢迎使用 AFC-7 (Intelligent Flight Controller) 专属的 Linux 板级支持包 (BSP)。本指南将协助您在 1 分钟内完成底层硬件资源的重分配，为 M33 飞控核心打造纯净的实时运行环境。

### 📁 目录文件清单
在执行安装前，请确保当前目录下包含以下核心文件：

AFC7LinuxBSPSetUp.sh：一键自动化部署脚本。

myb-afc7-2GB.dtb：AFC-7 专属编译的底层设备树固件。

myb-stm32mp257x-2GB_extlinux.conf：双启动引导配置文件（防变砖安全网）。

issue：AFC-7 专属的终端欢迎界面配置。

check_afc7_hw.sh：硬件资源全景巡检脚本。

### 🛠️ 第一步：一键极简安装 (在电脑 WSL2 / Linux 端执行)
请在您的电脑终端中，进入本 LinuxBSP 目录。
(注意：请先用文本编辑器打开 AFC7LinuxBSPSetUp.sh，确认里面的 TARGET_IP 为您开发板当前的实际 IP 地址。默认为 192.168.0.10)

无需更改任何文件权限，直接执行以下命令：

```Bash
bash AFC7LinuxBSPSetUp.sh
```
脚本将自动完成所有文件的安全推送，并在下发指令后自动重启开发板。当终端提示“部署指令下发完毕”时，您即可将目光转向开发板的串口终端屏幕。

### ✅ 第二步：测试与验证 (在开发板串口终端执行)
开发板重启后，请通过串口工具（如 MobaXterm）观察系统启动过程，并进行以下两项核心验收：

#### (1) 检查 AFC-7 专属品牌欢迎页
在系统内核跑码结束后，弹出 login: 登录提示符之前，请观察屏幕是否输出了如下极具辨识度的 ASCII 欢迎界面：

```Plaintext
=================================================================
   ___   _________   ____
  / _ | / __/ ___/  /_  /
 / __ |/ _// /__     / / 
/_/ |_/_/  ___/    /_/  
                         
 AFC-7 Intelligent Flight Controller
 Copyright (C) Zhuofei Zhikong Technology Co., Ltd.
 Chief Designer: Qinghua Zeng
 Version: V1.01
 Date: 2026.05.07
=================================================================
```

验收标准： 如果成功看到此界面，说明基础文件推送与系统引导已完美接管。请输入用户名 root 登录系统。

#### (2) 运行硬件全景巡检脚本
登录系统后，我们需要确认 A35 内核是否已经按照预期，将所有的飞控核心外设（串口、PWM、I2C1/3、SPI 等）物理控制权释放给了 M33。

一键安装脚本已提前将巡检工具存放至主目录。请直接执行以下命令：

```Bash
cd /root
./check_afc7_hw.sh
```
验收标准：
请仔细逐项核对终端输出的结果：

**[串口]：** 仅保留 ttySTM0，不能出现 ttySTM1/3/4/6/8/9。

**[硬实时接口]：** SPI 主机目录应为空；PWM 目录应为空；无飞控专属 ADC 节点；I2C 无 1 和 3。

**[A35 机载盘]：** USB Root Hub 及摄像头/硬件解码节点 (video0~video6) 正常挂载。

🎉 恭喜！ 如果巡检项目全部绿灯通过，标志着 AFC-7 双核异构系统的控制权边界已正式确立。
